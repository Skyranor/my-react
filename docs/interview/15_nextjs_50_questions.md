# 🚀 Модуль 15: 50 Вопросов и Ответов по Next.js (Ultimate Guide)

Этот справочник разбит на **5 категорий по 10 вопросов**. Ответы даны с учетом **App Router (Next 13-15)**, архитектурных подходов и практических советов (Pitch).

---

## 🏗️ Блок 1: Рендеринг и Компоненты (1-10)

### ❓ 1. В чем главное отличие Server Components (RSC) от SSR?
*   **Ответ:** SSR рендерит HTML на сервере, но отправляет весь JS-код компонентов в браузер для гидратации. RSC рендерится **только на сервере** и вообще не отправляет JS-код в браузер.
*   **Pitch:** *"RSC уменьшают размер бандла до 0 КБ для статических частей, а SSR просто ускоряет первую загрузку."*

### ❓ 2. Как избавиться от ошибки "window is not defined" при рендеринге?
*   **Ответ:** Проблема в гидратации (Server рендерит код, где есть `window`).
*   **Решение:** Обернуть вызов в `useEffect`, проверить `typeof window !== 'undefined'`, или использовать `next/dynamic` с опцией `{ ssr: false }`.

### ❓ 3. Что такое Partial Prerendering (PPR)?
*   **Ответ:** Комбинация статики и динамики на одной странице. Сервер моментально отдает статический каркас из CDN, а динамические зоны `<Suspense>` досылает параллельно через Streaming.

### ❓ 4. Когда использовать `@template.js`, а когда `@layout.js`?
*   **Ответ:** `Layout` сохраняет состояние (State) и не перерендерится при переходах. `Template` монтируется **заново** на каждый переход.
*   **Пример:** `Template` нужен для анимаций входа/выхода страниц или сброса форм.

### ❓ 5. Как заставить маршрут быть динамическим (SSR)?
*   **Ответ:** Поставить `export const dynamic = 'force-dynamic'` или вызвать чтение динамических данных: `await cookies()`, `await headers()`.

### ❓ 6. Что такое Streaming в Next.js?
*   **Ответ:** Потоковая передача HTML кусками (chunks). Сначала рендерится статика, тяжелые блоки досылаются асинхронно по одной HTTP-сессии.

### ❓ 7. Можно ли использовать `"use client"` внутри Server Component?
*   **Ответ:** Да. Но не как импорт вложенности. Нужно передавать Client Component как `children` внутрь Server Component, чтобы сохранить серверный рендеринг предка.

### ❓ 8. Что такое `use()` в React 19 / Next 15?
*   **Ответ:** Новый хук, разворачивающий `Promise` прямо внутри рендер-функции. Его можно вызывать условно (внутри `if`), в отличие от стандартных хуков.

### ❓ 9. Зачем нужен `unstable_noStore`?
*   **Ответ:** Декларативный способ отключить кэширование конкретного вызова данных внутри компонента, переводя его в динамический рендеринг.

### ❓ 10. Как исправить Flash of Unhydrated Content (FOUC)?
*   **Ответ:** Использовать `<Suspense>` для тяжелых данных и жестко фиксировать `width/height` или скелетоны, чтобы дерево DOM не прыгало при оживании React.

## 📦 Блок 2: Data Fetching и Кэширование (11-20)

### ❓ 11. Как работает дедупликация запросов fetch()?
*   **Ответ:** Если 5 компонентов вызывают `fetch('api')`, Next.js склеит их в 1 запрос на сервере. Работает только для `GET` запросов внутри одного дерева рендеринга.

### ❓ 12. Как настроить бесконечный кэш вызова `fetch`?
*   **Ответ:** `fetch(url, { cache: 'force-cache' })`.

### ❓ 13. Как настроить ISR (регенерацию) для одного fetch-запроса?
*   **Ответ:** `fetch(url, { next: { revalidate: 60 } })`. Обновит данные в фоне.

### ❓ 14. Как удалить данные из кэша по тегу (On-demand Revalidation)?
*   **Ответ:** Поставить `fetch(url, { next: { tags: ['posts'] } })` и вызвать `revalidateTag('posts')` в Server Action.

### ❓ 15. Что такое Server Actions и как они работают?
*   **Ответ:** Функции с директивой `"use server"`. При их вызове Next.js сам генерирует скрытый `POST` запрос.
*   **Pitch:** *"Это заменяет создание кастомных API-роутов для мутаций данных."*

### ❓ 16. Зачем нужна функция `cache()` из React в Next.js?
*   **Ответ:** Для дедупликации функций, которые **НЕ используют fetch** (например, SQL-запросы через Prisma). `const getCachedUser = cache(getUserFromDB)`.

### ❓ 17. Как затипизировать ответ формы в Server Actions?
*   **Ответ:** Использовать `useFormState` (или `useActionState` в Next 15) вместе с библиотекой валидации `Zod`.

### ❓ 18. Как получить Cookies в Server Action?
*   **Ответ:** Вызвать функцию `cookies()` из `next/headers`.

### ❓ 19. Как отобразить оптимистичное обновление в Next.js?
*   **Ответ:** Хук `useOptimistic`. Сначала рендерим «нарисованный» результат, а после ответа бэкенда синхронизируем с настоящим.

### ❓ 20. Куда кидать ошибки при неудачном fetch?
*   **Ответ:** Выкинуть `throw new Error()`. Next.js перехватит её на уровне файла `error.js` в корне папки роута.

## 🧭 Блок 3: Роутинг и Middleware (21-30)

### ❓ 21. В чем разница между `[id]` и `[[...id]]`?
*   **Ответ:**
    *   `[id]` — обязательный параметр (`/user/1`).
    *   `[...id]` — Catch-all. Поймает `/user/1/2/3`.
    *   `[[...id]]` — Optional Catch-all. Поймает и `/user` (без параметров).

### 22. Что такое Parallel Routes (Параллельные маршруты)?
*   **Ответ:** Папки вида `@analytics` и `@feed`. Позволяют рендерить два независимых экрана (например, Дашборд и Чат) на одном макете с разными `error.js` и `loading.js`.

### ❓ 23. Что такое Intercepting Routes (Перехватывающие маршруты)?
*   **Ответ:** Папка вида `(.)photo`. Позволяет открыть страницу как «Модалку» при клике на странице, но при прямой ссылке `/photo` откроется полноценная страница. (Как просмотр фото в Instagram).

### ❓ 24. В чем разница между `Redirect` и `Rewrite` в `next.config.js`?
*   **Ответ:**
    *   `Redirect` — меняет URL в браузере (301/302 код).
    *   `Rewrite` — мапит URL на другой ресурс скрытно, URL в браузере не меняется (как прокси).

### ❓ 25. Что такое Multi-Zones?
*   **Ответ:** Возможность собрать один сайт из независимых Next.js приложений (например, `/blog` — это одно приложение Node, `/buy` — другое), склеенных через Rewrite.

### ❓ 26. Как получить параметры запроса (Query Params) в Server Component?
*   **Ответ:** Через пропс `searchParams` функции страницы: `async function Page({ searchParams })`.

### ❓ 27. Какие ограничения среды Edge Runtime для Middleware?
*   **Ответ:** В Middleware нельзя работать с файловой системой Node (`fs`) и тяжелыми крипто-модулями. Он запускается на базе движка V8 без Node.js API.

### ❓ 28. Как программно сделать редирект на сервере?
*   **Ответ:** Вызвать функцию `redirect('/dashboard')` из `next/navigation`.

### ❓ 29. Зачем банку динамический роутинг?
*   **Ответ:** Для отображения истории конкретной карты по маске `/cards/[cardId]/history`.

### ❓ 30. Как сделать страницу 404 кастомной в App Router?
*   **Ответ:** Создать файл `not-found.js` в папке роутинга. Вызывается функцией `notFound()`.

## ⚡ Блок 4: Производительность и Оптимизации (31-40)

### ❓ 31. Какая проблема `<Image>` в Next.js убирает Cumulative Layout Shift (CLS)?
*   **Ответ:** Обязательность полей `width` и `height` (или `layout="fill"`). Браузер резервирует место под картинку **до** её скачивания.

### ❓ 32. Как настроить ленивую загрузку компонентов (Lazy loading)?
*   **Ответ:** `const Component = dynamic(() => import('./Item'), { ssr: false })`.

### ❓ 33. Зачем оптимизировать шрифты через `next/font`?
*   **Ответ:** Он скачивает шрифты на этапе билда и обслуживает локально. Нулевой Flash of Unstyled Text (FOUT).

### ❓ 34. Как `<Link>` подгружает страницы до клика (Prefetching)?
*   **Ответ:** Как только `<Link>` попадает во viewport браузера, Next.js подкачивает `Page.js` в рантайм кэш. Переход будет мгновенным (0мс).

### ❓ 35. Системный анализатор бандлов. Как проанализировать вес сборки?
*   **Ответ:** Использовать плагин `@next/bundle-analyzer`. Он генерирует визуальную карту веса JS файлов.

### ❓ 36. Как оптимизировать сторонние скрипты?
*   **Ответ:** Через `<Script src="..." strategy="lazyOnload" />`. Стратегия говорит браузеру грузить скрипт только после полной готовности страницы (например, Яндекс.Метрику).

### ❓ 37. Как генерировать динамические метатеги для SEO?
*   **Ответ:** Функция `export async function generateMetadata({ params })`.

### ❓ 38. Как подгружать CSS только для нужного роута?
*   **Ответ:** Использовать CSS Modules (`.module.css`). На странице подгрузится только CSS текущих компонентов, CSS других страниц будет отсечен.

### ❓ 39. В чем разница между `priority` и `loading="lazy"` в `<Image>`?
*   **Ответ:** флаг `priority` сообщает браузеру, что это приоритетное изображение (LCP). Его нужно загрузить в первую очередь без lazy load.

### ❓ 40. Что такое `next/navigation` `useRouter`?
*   **Ответ:** Клиентский хук навигации (`push`, `replace`, `back`). Работает только внутри `"use client"`.

## 🛡️ Блок 5: Безопасность и Инфраструктура (41-50)

### ❓ 41. Как защитить Server Actions от CSRF?
*   **Ответ:** Next.js защищает их из коробки, генерируя внутренние токены безопасности на каждый POST вызов формы.

### ❓ 42. Безопасно ли хранить секреты в `.env` в Next.js?
*   **Ответ:** Да. Секреты доступны только на уровне Node.js (Сервер). В браузер попадают только переменные с префиксом `NEXT_PUBLIC_`.

### ❓ 43. Как организовать деплой Next.js на обычную VPS (не Vercel)?
*   **Ответ:** Через **Docker**. Собирается контейнер с Node.js, запускается `npm run start`. Кэширование настраивается через Redis.

### ❓ 44. Какую проблему решает подход Static Exports (`output: 'export'`)?
*   **Ответ:** Генерирует чисто статический HTML/JS/CSS без Node.js сервера под капотом. Можно разместить на AWS S3 или GitHub Pages. Минус — нет SSR и ISR.

### ❓ 45. Какую БД лучше всего использовать с Next.js?
*   **Ответ:** Next.js атомарен к базам. Но для бессерверных сред отлично подходят PostgreSQL через Prisma/Drizzle.

### ❓ 46. Как в Next.js использовать логирование на проде (Sentry)?
*   **Ответ:** Поставить пакет `@sentry/nextjs`. Он автоматически интегрируется в роутинг и отслеживает падения как на сервере, так и на клиенте.

### ❓ 47. Можно ли использовать WebSockets в Next.js?
*   **Ответ:** Да, но не через обычный `next start`. Придется поднять кастомный сервер `server.js` (на базе Express), чтобы привязать Socket.io к тому же порту.

### ❓ 48. Как настроить HTTPS локально в Next.js?
*   **Ответ:** Использовать утилиту `mkcert`, или запустить `next dev --experimental-https` в последних версиях.

### ❓ 49. Что такое «Zod Validation» в API роутах?
*   **Ответ:** Валидация типов данных десериализованного `req.json()`. Предохраняет от SQL-инъекций и невалидных структур.

### ❓ 50. Какая метрика важнее всего для интернет-банкинга?
*   **Ответ:** **FID (First Input Delay)** или **INP (Interaction to Next Paint)**. Как быстро приложение отреагирует на клик «Оплатить». Next.js оптимизирует это за счет быстрой гидратации.
