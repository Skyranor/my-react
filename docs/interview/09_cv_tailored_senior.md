## 🏛️ Блок 1: Опыт в Сбербанке (Микрофронтенды и BDUI)

### ❓ В: "Как вы организовали Backend-Driven UI (BDUI) для Вкладов? Как версионировать контракты?"
*   **Официальный подход:**
    BDUI — это когда бэкенд присылает JSON с описанием компонентов: `[{ type: "input", name: "sum" }, { type: "button", text: "Открыть" }]`.
*   **🗣️ Pitch (Опора на ваше CV):**
    > *"В Сбербанке для модуля «Вклады» мы использовали BDUI. Главная проблема здесь — **стабильность контракта**. Чтобы избежать падений UI, я внедрил **runtime-валидацию через Zod**. Если бэкенд присылал новое поле или менял тип, Zod ловил это на входе, и мы показывали безопасный Fallback («Обновите приложение» или заглушку), вместо «белого экрана». Для версионирования мы жестко фиксировали типы схем на каждый релиз."*

---

### ❓ В: "Как изолированные микрофронтенды общаются между собой в банке (Event Layer)?"
*   **Официальный подход:** Использование шины событий (Event Bus), Custom DOM Events или единой платформы хоста.
*   **🗣️ Pitch:**
    > *"Микрофронтенды не должны знать друг о друге напрямую (`no hard imports`). Я проектировал **Event-management слой**. Мы использовали стандартный `EventTarget` или `BroadcastChannel` (для вкладок браузера). Один микрофронт (например, Профиль) кидает событие `BALANCE_UPDATED`, другой (Вклады) ловит его и обновляет свой внутренний стейт. Это гарантирует ортогональность модулей."*

## 📱 Блок 2: Опыт в Ainari (React Native & AI)

### ❓ В: "Чем New Architecture в React Native отличается от Old? Зачем вам это было нужно в Kaginari?"
*   **Официальный ответ:**
    *   **Old:** Связь JS и Native идет через **Bridge** (асинхронно по JSON).
    *   **New:** Работает через **JSI (JavaScript Interface)**. JS держит прямую ссылку на методы C++ нативных модулей. Сверху накручены **Fabric** (новый рендерер) и **TurboModules** (ленивая загрузка модулей).
*   **🗣️ Pitch:**
    > *"В приложении Kaginari я выбрал New Architecture (Expo). Главная боль старого Bridge — это асинхронность. Если вам нужно делать real-time обработку звука (Text-to-Speech), задержки бандла JSON губительны. **JSI** позволяет вызывать нативные методы напрямую без сериализации, что критически снизило latency для наших AI-голосовых функций."*

---

### ❓ В: "Что такое Generative UI на базе LLM Function Calling? Как это безопасно рендерить?"
*   **Официальный подход:** LLM не возвращает JSX напрямую. Она возвращает JSON-аргументы для функций, а фронтенд рендерит заготовленный виджет.
*   **🗣️ Pitch:**
    > *"В Kaginari мы не скармливали пользователю опасный HTML от нейросети. Мы использовали **Function Calling**. LLM решала: «Сейчас я хочу показать виджет словарной карточки» и возвращала объект `{ widget: 'Card', data: { word: 'hello' } }`. Моя архитектура мапила этот JSON на строго типизированный React-компонент `<VocabularyCard />`. Это безопасно, быстро и гарантирует дизайн-код приложения."*

## 📐 Блок 3: Архитектура (Feature-Sliced Design)

### ❓ В: "Почему вы выбрали Feature-Sliced Design (FSD)? Какие минусы у этой архитектуры?"
*   **Официальный подход:** FSD делит проект на слои: `app`, `pages`, `widgets`, `features`, `entities`, `shared`. Правило: нижний слой не может импортировать верхний.
*   **🗣️ Pitch:**
    > *"FSD идеален для командной работы. Разработчик сразу видит, где лежит бизнес-логика (`features`), а где структуры данных (`entities`). 
    > **Главный минус:** Высокий порог входа для джунов и овербандл папок при первых этапах. Но для банка, где важна масштабируемость и нет места спагетти-коду, FSD окупается за 1-2 спринта."*

### 🛡️ Убойный аргумент для Альфа-Банка:

Поскольку вы делали **миграцию React 17 -> 18 в Сбербанке**, вас точно спросят: *"Что создавало проблемы в миграции?"*.
*   **Ваша заготовка:**
    > *"Самое сложное — это **автоматический батчинг (Automatic Batching)** в React 18. Теперь он батчит и асинхронные обработчики (`fetch`, `setTimeout`). В паре мест у нас логика завязывалась на то, что стейт мутирует пошагово. Пришлось точечно использовать `flushSync()`. Также `Strict Mode` в 18 версии монтирует компоненты дважды в деве, что скрыло пару старых утечек памяти в наших микрофронтендах."*
