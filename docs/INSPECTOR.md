# Документация по React Fiber Inspector

**Fiber Inspector** — это встроенная панель инструментов (DevTools) для отладки вашего кастомного React-фреймворка. Она позволяет визуализировать дерево компонентов, отслеживать фазы рендеринга и профилировать производительность.

---

## 🌳 Вкладка "Trees" (Дерево Fiber)

Эта вкладка разделена на две панели:
1.  **CURRENT DOM TREE** — Текущее состояние дерева, которое отображено в браузере.
2.  **WORK-IN-PROGRESS** — Временное дерево, на котором выполняются расчеты перед тем, как закоммитить изменения в DOM.

### 🔢 Что означают цифры и значки?

*   **`x2`, `x3`, ... (Желтый текст)** — **Счетчик рендеров**. Показывает, сколько раз компонент **реально выполнил** свою функцию рендера (вызвал JS-код).
*   **`(Skip: N)` (Серый текст)** — **Счетчик пропусков (Bailout)**. Показывает, сколько раз запуск функции был **пропущен** благодаря оптимизации (когда props и state не изменились).
*   **`+` (Зеленый бейдж)** — **PLACEMENT**. Элемент был только что создан и добавлен в дерево.
*   **`↻` (Синий бейдж)** — **UPDATE**. Элемент обновил свои свойства (Props) или состояние (State).
*   **`-` (Красный бейдж)** — **DELETION**. Элемент удален из дерева.

### 🎨 Стилизация тегов:
*   <span style="color: #61dafb; font-weight: bold;">Голубой жирный (`<App>`)</span> — **Функциональные компоненты**.
*   <span style="color: #98c379;">Зеленый (`<div>`)</span> — **Стандартные HTML-элементы**.
*   <span style="color: #5c6370;">Серый (`<TEXT_ELEMENT>`)</span> — **Текстовые узлы**.

---

## ⏱️ Вкладка "Profiler" (Производительность)

Содержит таймлайн всех коммитов, помогая найти "тяжелые" рендеры.

### 📊 Показатели:
*   **`Commit #N [Mount / Update]`** — Номер фиксации изменений и ее тип.
*   **`Total: N ms`** — Суммарное время, затраченное на фазы `Render` и `Commit` для этого обновления.
*   **График (Bar)** — Визуальная длина показывает пропорциональное время относительно самого тяжелого коммита.

---

## 🐢 Дополнительные функции

*   **Slow Motion**: Переключатель в правом верхнем углу. Искусственно добавляет задержку в 500ms между шагами фазы Commit. Позволяет визуально "увидеть", как React обходит дерево и вставляет элементы по одному (особенно полезно для отладки анимаций и очередности).
*   **Подсветка (Highlighter)**: Наведение на узел в доке подсвечивает его физический аналог на самой странице голубой рамкой.
*   **Статус (IDLE / RENDER / COMMIT)** — Позволяет в реальном времени видеть, в какой фазе сейчас находится WorkLoop.
